LAB 1 - Let's GIT to work

Wat is GIT en hoe moeten we er mee werken? Dit hield de eerste opdracht voor Webtech 2 in. In groepjes van 3 of 4 moesten we samen bijdragen aan een repository. Eén iemand moest de repository aanmaken na het installeren van GitHub.

Basis Git commando's:

  git clone: clones/kopieert de repository map op je computer.
  git branch: toont lijst van alle branches en op welke branch je zit (default is master).
  git checkout -b (nieuwe branch): creert nieuwe branch en wisselt er meteen naar.
  git checkout: wisselen van branch.
  git merge (nieuwe branch): merge eigen branch met master.
  git branch -d (nieuwe branch): deleten van eigen branch.
  git add (bestandsnaam): enkel bestand toevoegen.
  git add (foldernaam)/ : een hele folder toevoegen.
  git commit -m: de bestanden of veranderingen committen met message.
  git push: de veranderingen/bestanden pushen naar de repository online.
  git pull: de lokale repository up-to-date brengen met de online versie.

Git commando's ivm forked repositories:

Als je wil bijdragen aan een repository van een andere git-gebruiker kun je zijn repository forken. Daardoor komt de repo ook in jouw repository overzicht te recht. 

  git clone: clones/kopieert de repository map op je computer.
  git remote add upstream (link van geforked repo): upstream wijst naar de originele repo, zo kun je altijd jouw versie updaten.
  git fetch upstream: haalt de veranderingen in de originele repo op.
  git merge upstream/master: alles samen voegen.
  git push origin master: pushen naar jouw forked repo.

Daarna kan je een pull request om jouw veranderingen toe te voegen aan de originele repository.
Je kan ook bijdragen aan een repository door jouw veranderingen toe te voegen onder een eigen branch en vanuit je eigen branch dan een pull request te sturen naar de originele repo. 

  Je doet git clone, git remote add upstream, git fetch upstream.
  git checkout -b (nieuwe branch): creëren van nieuwe branch en er meteen naar wisselen.
  git add --all: alle veranderingen vn de repo op je persoonlijke branch toevoegen.
  git commit -m: de toevoegingen committen met message (zonder geeft error).
  git push origin (nieuwe branch): maakt eigen branch aan online op GitHub zelf en pusht er jouw veranderingen naar.

Opdracht 1 - gebruik van CSS3 transformations

  SYNTAX:
  transform : function(value) //;other transform functions//;
  transform-origin: left center; //van waaruit de transformatie moet gebeuren//

De CSS3 transform eigenschap zorgt er voor dat het uitzicht van het geselecteerde element gemanipuleerd wordt volgens bepaalde transformatie functies. Het geselecteerde element behoudt zijn oorspronkelijke positie en grootte welke transformatie ook wordt uitgevoerd.
Enkele voorbeelden van zulke transformatie functies zijn:

rotate(angle)

Roteert een element onder een bepaalde hoek.

    background-color: blue;
    width: 40px;
    height: 40px;

  }

  • scale(2,4): horizontaal vergroten (x) met 2, verticaal vergroten (y) met 4.
  • scaleX(value) en scaleY(value): meer specifiek vergroten voor afzonderlijk de X- of Y-as.

    SYNTAX:
  • skew(x,y): schuintrekken volges x- of y-as in één functie.

Opdracht 2 - gebruik van CSS3 animation

  SYNTAX
  Animatie waarbij de eigenschappen apart worden aangeroepen:
  #box 
  {
    animation-name: bounce;
    animation-duration: 4s;
    animation-iteration-count: 10;
    animation-direction: alternate;
    animation-timing-function: ease-out;
    animation-fill-mode: forwards;
    animation-delay: 2s;
  }

  Animatie genoteerd in 1 lijn:
  animation: name duration iteration-count direction timing-function fille-mode delay;
  animation: bounc 4s 10 alternate ease-out forwards 2s;
Waarvoor staan al deze animatie-eigenschappen?
  • name: naam van de animatie.
  • duration: hoelang de animatie zal duren.
  • iteration-count: hoeveel keer de animatie herhaalt moet worden.
  • direction: in welke richting de animatie moet afgespeeld worden bv: normal,reverse,...
  • timing-function: hoe de animatie moet verlopen over zijn duur bv: ease-in-out, lineair,...
  • fill-mode: bepaald waar de animatie moet stoppen.
  • delay: de vertraging waarmee de animatie moet starten.
Een animatie kan ook in verschillende stappen gebeuren.
  @keyframes [animation-name]{
  from{wat er geanimeerd moet worden}
  to{wat er geanimeerd moet worden}
  }

  OF

  @keyframes [animation-name]{
  0%{}//begin
  50%{}//midden
  100%{}//einde
  }
Animations en transformations combineren

  }

DRY

Don't Repeat Yourself

Tijdens het schrijven van Javascript code kom je vaak/soms op het punt dat je merkt dat je code herhaalt. Te veel herhaling in je code moet je proberen te vermijden. Houdt DRY daarom altijd in je achterhoofd.

Voorbeeld JavaScript geschreven met herhaling en DRY
  MET HERHALING

  $('#myCheckbox').click(function() {
    if (this.checked) {
      $('#field_foo').val(config.foo);
      $('#field_bar').val(config.bar);
      $('#field_baz').val(config.baz);
    } else {
      $('#field_foo').val('');
      $('#field_bar').val('');
      $('#field_baz').val('');
    }
  });  

  MET DRY IN JE ACHTERHOOFD

  $('#myCheckbox').click(function() {

    var checked = this.checked;

    $.each(['foo', 'bar', 'baz'], function(i,v) {
      if(checked)
      {
        $('#field_' + v).val(config[v]);
      }
      else
      {
        $('#field_' + v).val('');
      }
    });
  });

Functions zijn JS Classes

In JavaScript spreken we niet van classes zoals in PHP waar je eigenschappen, gegevens en functies in plaatst. Maar dit doen we met functions. 

  function Person(){}   
  var liesbeth = new Person()
  liesbeth is hier een nieuwe Person op dit moment nog zonder eigenschappen.

  function Person(lastname, firstname, age){
    this.lastname = lastname,
    this.firstname = firstname,
    this.age = age
  }  
  var liesbeth = new Person(Vanaerschot, Liesbeth, 25)
  console.log(liesbeth.firstname + " is " + liesbeth.age + " jaar.");
  hier is de output "Liesbeth is 25 jaar."

Prototypical Inheritance

Om er bijvoorbeeld voor te kunnen zorgen dat een object de eigenschappen van een ander object kan overnemen/gebruiken bouwen we in JavaScript het overerving principe van classes na door middel van prototypes. Prototypical inheritance maakt het coderen flexibeler en hierdoor hou je rekening met DRY. 

  voort bouwen op de Person function:
  
  Person.prototype.sayHello = function(){
    alert( this.firstname + " says hello!");
  }
  liesbeth.sayHello();
  output: Liesbeth says hello!

  var Child = function(){
    Person.call(this);
  }
  Child.prototype = new Person(); 
  Child.prototype.constructor = Child;
  Child.prototype.walk = function(){
    alert('I can walk!');
  }     

  var elle = new Child(Desmedt, Elle, 1);
  elle.sayHello();
  elle.walk();
  output: Elle says hello! en I can walk!

AJAX en API

1. AJAX

AJAX staat voor Asynchronous Javascript And XML, maar in plaats van XML gebruiken wij JSON (JavaScript Object Notation). De resultaten die we terug krijgen van een AJAX-call zullen objecten zijn. Soms veranderen ze AJAX dan in AJAJ...
Wat doet AJAX nu eigenlijk?
AJAX zorgt er voor dat het mogelijk is om een asynchronische webapp te kunnen bouwen. In plaats van dat verschillende functies achter elkaar uitgevoerd moeten worden kan je met AJAX bekomen dat ook al is er één proces reeds gestart een ander proces ook uitgevoerd kan worden. Met AJAX kan je gegevens van een server of API oproepen (of saven naar een server) zonder dat na elk proces de webpagina moet herladen. De gebruiken merkt dus niets van wat er in de achtergrond uitgevoerd wordt. Hij krijgt enkel zonder page reload het bekomen resultaat te zien. Met AJAX herlaad maar een stukje van de webpagina: bijvoorbeeld enkel de div waar je het resultaat in wilt weergeven. 

  var currentID = $(this).parent().attr('id');

  $.ajax({
      type: 'POST',
      url: 'http://localhost:3000/order',
      data: {_id: currentID},
      dataType: 'json',
    })
   .done(function(data){
      console.log('success');
      console.log(data);
      return previd = data;
   });
2. API's

API staat voor Application Programming Interface. Public API's zijn een soort van bibliotheken die veel informatie bevatten in de vorm van JSON. Als je werkt met API's dan zijn de AJAX calls die je er naartoe stuurt enkel GET verzoeken en geen POST zoals in mijn voorgaande voorbeeld. Aan de meeste publieke API's kan je zelf niets toevoegen, maar wel informatie uithalen.
Hieronder de getForecast functie om het weer op te halen aan de hand van de coördinaten van jouw huidige positie: 

  weer.prototype.getForecast = function()
    {
      var his = this;
      var latitude = localStorage.getItem("latitude");
      var longitude = localStorage.getItem("longitude");
      console.log(latitude + " " + longitude);
      var apiKey = "uw apikey";
      var url = "https://api.forecast.io/forecast/" + apiKey +"/" + latitude + "," + longitude;
      $.ajax(url, {
          dataType: "jsonp"})
          .done(function(data)
              {
                his.setForecast();
                console.log("succes");

    }

Een eigen API key verkrijg je door een account aan te maken voor de API die je wilt gebruiken. Zonder kan je geen connectie maken. In de .done() methode krijg je het resultaat terug waar in je dan zelf kunt bepalen wat je doet met het resultaat. De longitude en latitude coördinaten bekom je door de methoden te gebruiken van de HTML5 Geolocation API: Geolocation DEMO. 

  if(navigator.geolocation)

LAB 6 - NODE.js

In LAB6 leerden we gebruik maken van het softwareplatform Node.js. De bedoeling was om een simpele questionapp uit te bouwen waar verschillende mensen korte vragen kunnen posten en kunnen stemmen op de reeds geposte vragen.

Wat is Node.js?

Door middel van het JavaScript platform Node.js kan je zowel client-side als server-side coderen in JavaScript. Waar je anders voor de server-side PHP nodig zou hebben, gebruik je met Node ook gewoon JavaScript. Dit heeft al één heel groot voordeel: 2 maal het gebruik van JavaScript werkt beter samen.
De applicaties die gebouwd worden met Node draaien op de Node server, die geïnstalleerd wordt wanneer je Node installeert op je computer. Hierdoor heb je voor het lokaal draaien van je apps bijvoorbeeld geen Apache meer nodig.
Door non-blocking io en het gebruik maken van een single thread, asynchronisch uitvoeren van functies en methoden wordt Node.js voornamelijk gebruikt om realtime applicaties uit te bouwen.

Non-blocking - Asynchronous I/O

Node en Express

Om wat duidelijke structuur in je Node-app te krijgen gebruiken we Express.js. Express.js is een webapplicatie framework speciaal voor Node.js. Wanneer je express installeert doe je dit best globaal zodat de Express structuur gebruikt kan worden voor alle Node-apps eendert waar op je computer.
Dependencies zoals Express.js installeer je door middel van npm install express in te voeren in de Node.js Command Prompt. En dan moet je Express nog toevoegen onder dependencies in je package.json file. Dit kan ook in 1 stap npm install express --save, door de save wordt Express meteen toegevoegd aan je dependencies. 

  PACKAGE.JSON bij start

  {
    {
    "name": "application-name",
    "version": "0.0.1",
    "private": true,
    "scripts": {
      "start": "node app.js"
    },
    "dependencies": {
      "express": "3.5.0",
      "jade": "*" 
      jade is de standaard template engine van express.
      hier in schrijf je html op een andere manier
    }
  }

  IN APP.JS

  (bovenaan niet vergeten bij te voegen.)
  var express = require('express');

Een Node app wordt door Express opgedeeld in een bepaalde structuur: Public, Routes en Views.
1. Public
In de map public plaats je je JavaScript files, CSS files, Images,...
2. Routes
In de map Routes vindt je de JavaScript bestanden die de "route" van je app bepalen, dus op welke pagina wat getoond wordt.
3. Views
In Views staan je jade of html bestanden. Dit zijn de paginas die in je route files worden aangeroepen. Hierop render je de inhoud. Deze bestanden zijn letterlijk de view(het uitzicht) van je website.

Wat is Backbone.js?

Backbone.js is een Javascript framework dat structuur brengt in clientside applicaties. Bij pure JavaScript of het jQuery framework moet je eigenhandig een goede structuur in je code proberen te brengen wat bij complexe applicaties al snel een rommeltje kan worden. De visualisatie van je applicatie en de server hangen bij Javascript niet automatisch aan elkaar, je moet dit zelf aan elkaar schrijven. Je kunt elementen selecteren waarop je bepaalde functies kan toepassen die deze elementen dan beïnvloeden of veranderen. Als je dan ook nog eens data wilt aanroepen of opslagen moet je apart een Ajax call sturen naar je server. Bij Backbone.js ligt dit anders. Het brengt structuur in je applicatie, logica in je Javascript code. Met behulp van Backbone.js kan je je datastructuur scheiden van de visualisatie van je applicatie. Maar met behulp van de MVC structuur hangt de server, data en visualisatie aan elkaar waardoor je applicatie synchroon uitgevoerd kan worden.

Waarvoor staat MVC?

Bij Backbone.js staat MVC voor Models, Views en Collections. Een model kan je vergelijken met een klasse zoals in PHP, maar een opgevulde klasse. Daarmee wordt er bedoeld dat een model gegevens/data bevat bijvoorbeeld: model friend kan voornaam, naam, adres,… van een vriend bevatten. Een model is dus een beter gestructureerd object, dat naast data ook nog eens methoden of functies kan bevatten. 

  MODEL VOORBEELD  

  var friendModel = Backbone.Model.extend({
    //setting default values
    Defaults :
    {
      firstname: 'Anonymous',
      lastname: 'Person',
      phone: 'no phone number',
      email: 'no email'
    }
  }); 

  var liesbeth = new friendModel(); 
  //wanneer je geen properties meegeeft (.set) krijg je de default values wanneer je liesbeth.get('firstname') doet.
  //liesbeth.get('firstname') zal 'Anonymous' weergeven.

  dus doe je best:
  var liesbeth = new friendModel();
  liesbeth.set({firstname: 'Liesbeth', 
                lastname: 'Vanaerschot', 
                email: 'lies.vanaerschot@gmail.com'});
  liesbeth.get('email') //returns lies.vanaerschot@gmail.com
  liesbeth.get('phone') //returns default: no phone number

De meeste mensen onder ons hebben natuurlijk meerdere vrienden, dus meerdere friend models. Maar het is moeilijk om bijvoorbeeld van al je vrienden hun voornaam op te roepen wanneer al die friend models losse stukjes zijn. Om dit makkelijker te maken hebben we een collection nodig. Een collection bevat letterlijk een collectie models, dus friend models zitten in de collection friends. Zo een collection kan je vergelijken met een array, maar krachtiger want in een collection kan je ook weer methoden en functies steken. 

  COLLECTION VOORBEELD  

  var FriendsCollection = Backbone.Collection.extend(
  {model: friendModel});

  var friends = new FriendsCollection();
  friends.set([liesbeth,other models]);

Hoe toon je nu al deze gestructureerde data aan de gebruiker? Hiervoor creëer je een view waarin je schrijft welk DOM element je wilt gebruiken om de data, die in de door jouw gecreëerde collection of model zit, te renderen.

  var FriendView = Backbone.View.extend({
    el: 'li',
    initialize: function(){
      this.render();
    },
    render: function(){
      this.$el.html(this.model.get('firstname'));
      return this;
    }
  }); 

  var friendview = new FriendView({model: liesbeth});
